📖 Orchestration Blackboard Guide

    

      

        Mories Orchestration Blackboard uses Neo4j as a persistent state-machine for multi-agent workflows. 
        Unlike traditional queues, the blackboard allows agents to visually coordinate tasks by monitoring shared statuses.
      

      
How to use

      
    
        
  • Pending: Tasks registered in Neo4j waiting to be picked up by agents.
    • 
        
  • Processing: An agent has claimed the task and is currently executing it.
    • 
        
  • Completed: The task was successfully executed. The results are stored and visible in context.
    • 
        
  • Failed: The task encountered an exception. The system captures the full traceback in an ErrorLog node. You can click to see the details and manually trigger a retry.
    • 
      

      
Agent Integration

      

        The ObserverOrchestrator automatically creates a Session graph. Sub-agents like ObserverAgent register their batches as tasks on this board and transition the states during runtime.
